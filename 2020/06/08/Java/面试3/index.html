Spring中的Bean是什么时候实例化的

第一：如果使用BeanFactory作为Spring Bean的工厂类，则所有的Bean都是在第一次使用的时候进行实例化

第二：如果使用ApplicationContext作为Spring Bean的工厂类

  • 如果Bean的scope(作用域)是singleton，并且懒加载为false，则在ApplicationContext启动的时候就实例化该Bean(spring中Bean默认是单例并且延迟加载)
  • 如果Bean的scope是单例，并且懒加载为true，则在第一次使用该Bean时进行实例化
  • 如果Bean的scope是prototype，也是在第一次使用该Bean时进行实例化

Spring Bean/创建出来的对象的作用域有哪些

  • singleton

Spring IOC容器只会创建该Bean定义的唯一实例

  • prototype

对于每一次从容器中调用Bean时（将其注入到另一个Bean中，或者以程序的方式调用容器的getBean方法）都会产生一个新的bean实例，但是Spring不能对一个prototype bean的整个生命周期负责，当容器初始化，配置万一个prototype实例后，就对该实例不闻不问了

  • request

每一次http请求都会产生一个新的Bean，同时该Bean只对当前http request有效

  • session

每一次http请求都会产生一个新的Bean，而且该Bean只对当前http session有效，不同session不同bean

  • global session

当一个bean的作用域为Global Session，表示在一个全局的HTTP Session中，一个bean定义对应一个实例。典型情况下，仅在使用portlet context的时候有效

Spring Bean生命周期

Spring Bean的生命周期有实例化、属性赋值、初始化和销毁4个大阶段

实例化的时候默认是单例

然后对实例化的Bean进行IOC注入

在初始化阶段检查Aware的相关接口并设置相关依赖，如果Bean实现了BeanNameAware接口，则将Bean的id传给setBeanName方法

如果Bean实现了BeanFactoryAware接口，则将BeanFactory对象传给setBeanFactory方法

然后如果Bean实现了BeanPostProcesser接口，可以在Bean初始化前后进行一些处理

调用Bean自身定义的init方法（InitializingBean接口，配置文件中制定能够init-method属性），去做一些初始化相关的工作

最后是销毁Bean，如果Bean实现了DisposableBean接口，则调用其destroy方法，或者配置文件中制定能够destroy-method属性

当应用上下文被销毁时，Bean会被销毁

谈谈你对IOC和DI的理解，它们有什么区别

  • IOC是控制反转，将原本要手动创建对象的控制权交给Spring框架管理
  • DI是依赖注入，在Spring框架创建Bean对象时，动态的将依赖对象注入到Bean组件

区别是IOC负责对象的创建，而DI负责对象属性的注入

BeanFactory和ApplicationContext接口有什么区别

ApplicationContext接口继承BeanFactory接口，Spring的核心工厂是BeanFactory，采用延迟加载，而ApplicationContext在加载配置文件时就会初始化Bean。

ApplicationContext是对BeanFactory的扩展，它可以进行国际化处理、时间传递、以及bean的自动装配

spring配置bean实例化有哪些方式

  • 类构造器创建
  • 静态工厂创建
  • 实例工厂创建

Bean注入属性有哪几种方式

构造器注入和set方法注入

Spring的核心类有哪些，各有什么作用

  • BeanFactory：产生一个新的实例，可以实现单例模式
  • BeanWrapper：提供统一的get及set方法
  • ApplicationContext：提供框架的实现和BeanFactory的所有功能

对AOP的理解

AOP是一种面向切面编程的设计思想，它最大的一个作用就是降低了程序的耦合度，它可以把多个类共有的代码抽取到一个切片中，等到需要时再切入对象中去，也可以让我们只关注业务本身，而不去关注安全、日志、事务之类的东西。

AOP的专业术语

  • 增强：就是你想要实现的功能以及时间点，例如安全、事务、日志等
  • 织入：将切面代码插入到目标对象某个方法的过程
  • 连接点：spring允许增强的地方，例如某个方法调用前后、方法抛出异常后
  • 切点：实际被切面织入的方法
  • 切面：增强和切点的结合
  • 目标对象：切点和连接点所属的类
  • 引介：是一种特殊的增强，为类添加一些属性和方法
  • 代理：一个类别织入增强后，就产生了一个结果类，他是融合了原类和增强逻辑的代理类

AOP的应用场景

权限管理、日志记录、事务管理、错误处理、缓存、同步、性能优化等方面
